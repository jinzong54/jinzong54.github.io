软件体系结构

软件体系结构：B/S C/S

C/S

  • c/s：（client/Server） 例如QQ
  • 需要编写服务器
  • 缺点：软件更新是需要通过更新客服端和服务器端
  • 优点：安全性

B/S

  • B/S：（Browser/Server）
  • 优点：只需要编写服务器程序
  • 缺点：安全性较差

WEB资源

  • html ：静态资源：浏览器直接运行
  • JSP/Server ：动态资源：需要先转化为静态资源再传给浏览器
  • 访问web资源 ：协议名://域名/统一资源定位符（域名会被解析为ip地址）

web服务器

  • Tomcat（Apache）：当前应用较广javaweb的服务器，
  • JBoss : 支持JavaEE规范的，EJB容器，企业级的JavaBean
  • GlassFish ：Oracle所开发，JavaEE规范
  • Resin : 满足JavaEE规范
