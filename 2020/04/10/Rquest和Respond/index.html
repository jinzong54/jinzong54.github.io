Request和Resphond

大致流程

客户端向服务器接收请求，服务器将会为请求开辟一个新的线程，服务器将请求封装交成request（request是请求的载体），同时还会创建respond对象，如果需要还可发回给客户端。服务将request交给Servlet处理，将处理结果交给respond，重新发回客户端。

大致概念

ServletRequest—–》与协议无关的Request
ServletResponse——》与协议无关的Respond

API

  • sendError(int sc):
    Sends an error response to the client using the specified status code and clearing the buffer.
  • sendError(int sc, java.lang.String msg):
    Sends an error response to the client using the specified status.
  • setHeader(java.lang.String name, java.lang.String value):
    Sets a response header with the given name and value.
  • addHeader(java.lang.String name, java.lang.String value):设置多值响应头Adds a response header with the given name and value.
  • setDateHeader(java.lang.String name, long date)
    Sets a response header with the given name and date-value.

  • Enumeration getHeaders（String name），适用于多值头

  • getServerPort(): 返回端口号

  • getQueryString(): 获取参数部分。username=xxx&password=xxxx;
  • getRequestURI ：没有前面端口域名部分的
  • getRequestURL ：获取完整路径

请求转发和请求包含

RequestDispatcher rd = request.getRequestDispatcher(“/Myservlet”);
请求转发：rd.forward(request,response);由下一个Servlet完成留头不留体。浏览器只是对第一个Servlet请求，并没有对访问转发的第二个发送请求。

请求包含：rd.include(request,response);由两个Servet完成响应头都留

request域：域中只有方法，没有参数，一般是在需要，转发或包含才使用到

Servlet中三大与对象：request，session，application

reponse解决乱码问题

国际将iso8859-1（只支持拉丁文）规定服务器为默认的解码方式，发送给浏览器也是用iso8859-1解码成字节流后，发送给浏览器，浏览器的默认解码方式是GBK导致错误

  • response.setCharaterEncoding(“utf-8”) –只是设置发出去的为utf-8编码的字节流
  • response.setHeader(“Contebt-Type”,”text/html;charset=utf-8”);不仅告诉浏览器我的编码是utf-8，同时设置了我的编码方式是utf-8
  • response.setContenType(“text/html;charset=utf-8”)
