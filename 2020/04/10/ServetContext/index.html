ServletConetxt

概述

服务器会为每一个应用创建一个Servlet对象

  • ServletContext对象的创建是在服务器启动时创建的
  • ServletContext对象的销毁是在服务器关闭时完成的
    ServletContext对象的作用是在整个web应用的动态资源的之间的共享数据。在Aservlet中向ServletContext对象保存一个值，然后在Bservlet中就可以获得这个值

如何获取ServletContext

  • HttpServlet继承于GenericServlet，在GenericServlet中已经定义了config成员，可以直接获取config对象，然后从config再调用getServletContext()。同时GenericServlet也定义了getServletContext()方法，

域对象的功能

与对象就是为了在多个不同的Servlet传递数据。域对象拥有存数据和取数据的功能。

  • PageContext
  • ServletRequest
  • HttpSession
  • ServletContext
    所有域对象内部都有一个Map，用于存储数据

常用函数

  • getAttribute(): Returns the servlet container attribute with the given name, or null if there is no attribute by that name.
  • getAttributeNames(): Returns an Enumeration containing the attribute names available within this servlet context.
  • removeAttribute(): Removes the attribute with the given name from the servlet context.
