JSP

JSP（java server pages）来源

  • 作用：本质也是功能更加强大的Servlet
    • 对于Servlet:不适合设置html响应体，需要大量的response.getWriter().print(““)
  • 大致工作流程 ：
      1. 获取jsp发送过来的请求参数
      1. 处理请求，得到处理后的数据
      1. 把结构保存倒request域中
      1. 转发显示jsp
  • JSP的组成 ： html + java 脚本 +jsp指令
    • jsp无需创建即可使用对象，被称为9大内置对象，例如request对象，response对象
    • 3中常用脚本
    • <% … %> java代码片段常用；
    • <%=…%> java表达式，一般只能放一条语句；
    • <%!…%> 声明，用于创建类的成员变量和成语方法，一般用不着，但面试一般会考.一般是类中的东西.

生命周期问题

JSP页面的生命周期：如Servlet的周期
我们假设要访问的jsp页面是index.jsp。
首先，用户发出请求index.jsp；
服务器会判断是否是第一次请求：
如果是的话，JSP引擎会把该JSP文件转换成为一个Servlet，生成字节码文件，并执行_jspInit()；
如果不是的话，JSP将会访问已生成的字节码文件，解析执行_jspServlet()方法。
jspService()方法被调用来处理客户端的请求。对每一个请求，JSP引擎创建一个新的线程来处理该请求。如果有多个客户端同时请求该JSP文件，则JSP引擎会创建多个线程。每个客户端勇士请求该JSP文件，则JSP引擎会创建多个线程。每个客户端请求对应一个线程。以多线程方式执行可以大大降低对系统的资源需求，提高系统的并发量及响应时间。但也要注意多线程的编程带来的同步问题，由于该Servlet始终驻于内存，所以响应是非常快的。
