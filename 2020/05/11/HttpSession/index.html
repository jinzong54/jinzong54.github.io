HttpSession

HttpSession概述

  • HttpSession是由JavaWeb提供的，用来回话跟踪的类，session是服务器的对象，保存于服务器
  • HttpSession是三大域对象之一（request，session，application（ServletContext））
  • HttpSession底层是以来Cookie，或是URL重写

HttpSession的作用

  • 会话范围：会话范围是某个用户从首次访问服务器，到关闭浏览器
    • 会话是指一个用户对服务器的多次连贯性请求！所谓连贯性请求，就是该用户多次请求中间没有关闭服务器
  • 服务器会为每一个客户端创建一个session对象，session就好比客户端的账户，它们被服务器保存到一个map中，这map被称为session缓存
    • Servlet获取：HttpSession session = request.getSession（）；
    • JSP中，session是内置的，可以直接使用
  • 相关域方法
    • void setAttribute(String name,Objetc value)
    • Object getAttribute(String name)
    • void removeAttribute(String name)

      session原理

      在浏览器访问服务器的时候，服务器在本地创建一个cookie，里面储存的一个JSEESIONID，不创建新会话的时候，保存JSEESIONID的cookie一直活着（不活动默认的时间是30分钟）。
  • 服务器通过sessionid可以取出session里面的值
  • request.getSession(false):如果缓存中cookie-jsessionid
  • invalidate(),使session死去
  • request.getSession().isNew();查看是否为新
